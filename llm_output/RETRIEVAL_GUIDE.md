## Understanding the naming convention

| Rule | Detail |
|---|---|
| Separator | Every `/` in the original path is replaced by `__` |
| Extension | Preserved exactly as-is |
| Reversible | Replace `__` with `/` to recover the original path |

**Example:**
```
index.html  →  index.html
```

You can tell ChatGPT this rule directly and it will resolve filenames back to their
original paths on its own:

> "Files in this project use `__` as a folder separator. `src__auth__login.ts`
> means `src/auth/login.ts` in the original project."

## Tips for working with ChatGPT Projects

- **Upload this guide before any source files.** It primes ChatGPT with the naming
  convention and the full file map, so subsequent responses are more accurate.
- **Group uploads by feature.** Upload all files related to one module or task
  together rather than random individual files — ChatGPT reasons better with
  cohesive context.
- **Use file size to prioritise.** The `size` column in the index below helps you
  avoid uploading unexpectedly large files when a smaller, more focused file would
  answer your question.
- **You don't need `code_map.json` unless you want structured queries.** This guide
  is enough for most workflows. Upload `code_map.json` only if you want ChatGPT to
  answer questions like "list all files in the `src/api/` module".
- **Keep the project focused.** Remove stale files from the ChatGPT Project when you
  move to a different task — a leaner project produces sharper answers.
